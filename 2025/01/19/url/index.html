URL

[TOC]

一.url地址（web address）

1.概念：统一资源定位服务符–完整描述网页和资源地址的一个标识方法。URL是一种URI，它标识一个资源（可以是电脑本地或者互联网资源，常见于标识互联网资源），并指定对其进行操作或获取该资源的方法。

2.语法格式：

protocol://hostname[:port]/path/[parameters] [?query]]#fragment

组成：

http: // localhost: 8888 / hello

(protocol) (host name) (port) (path)

  • scheme:协议（https/http/ftp）
  • host: 服务器Ip或域名
  • port#: 服务器端口(协议默认端口，缺省(默认)端口80)
  • path: 访问资源路径，由零或多个“/”符号隔开的字符串，一般用来表示主机上的一个目录或文件地址。
  • query-string : 参数，发送给http服务器的数据；可有多个参数，用“&”符号隔开，每个参数的名和值用“=”符号隔开。
  • anchor: 锚(跳转到网页的指定锚点位置)#之后是锚点。

URL只是标识资源的位置，而HTTP是用来提交和获取资源。

二.uri概念

1.URI：统一资源标识符，是一个用于标识某一互联网资源的字符串。URI提供了一套简单可扩展的方式对资源进行标识，可以唯一地标识网络上的各种资源。

2.关系：

URI 可以认为是一个抽象的概念，所有的 URL 以及 URN 都是 URI。

URI 可以被分类成 locator 或者对应的名称表示，也就是包含了 URL 和 URN 的概念。因此，平常我们在说 URL 的时候，它其实也可以被称为 URI。

同样，URN 其实比较好区分开，在使用唯一标识资源名称时可以使用，但是 URI 和 URL 如何区分在哪个场景进行使用？

URI 不保证提供该资源的访问方式，或者隐含保证该资源是否存在（其实语义就是该 URI 就是一个名称表示），但是在上一段中又声明了URI 会被分类成name 或者 locator,表示 URI 应该包含locator 这种访问方式。

每个 URI 都需要包含有起始 scheme 名称。比如：https://www.example.com，这样的一串字符串就可以称为 URI，但是明确标识了应该如何去访问这个资源，同时它也是 URL，因为 URL 是用来告知接收方获取该资源的方式。

3.对URI进行编码

在 URI 里只能使用 ASCII 码，但如果要在 URI 里使用英语以外的其他语言该怎么办呢？

某些特殊的 URI，会在 path、query 里出现“@&?”等起界定符作用的字符，会导致 URI 解析错误，这时该怎么办？

所以，URI 引入了编码机制，对于 ASCII 码以外的字符集和特殊字符做一个特殊的操作，把它们转换成与 URI 语义不冲突的形式。这在 RFC 规范里称”转义”：

直接把非 ASCII 码或特殊字符转换成十六进制字节值，然后前面再加上一个“%”。

例如，空格被转义成“%20”，“?”被转义成“%3F”。而中文、日文等则通常使用 UTF-8 编码后再转义，例如“银河”会被转义成“%E9%93%B6%E6%B2%B3”。

有了这个编码规则，uri可以支持任意的字符集用任何语言来标记资源。

三.路由

1.概念：数据包从源地址传输到目的地址过程中所经过的路径选择机制。这一过程依赖于路由器设备上的特定算法来进行最优路径计算并转发报文。对于静态路由而言，管理员手动设置通往目标网络的具体下一跳IP地址或接口名称；而动态路由则依靠诸如RIPv2、OSPF等协议自动学习更新可达性信息。

2.工作原理：

数据以数据包的形式沿着任何网络传输。每个数据包都有一个标头，其中包含有关数据包预定目的地的信息。当数据包向目的地移动时，多台路由器可能会对其进行多次路由

当数据包到达时，路由器首先在路由表中查找其地址。类似通过查阅地图找到前往目的地的最佳路线。然后，路由器将数据包转发或移动到网络中的下一个点。

例如，当从办公室网络中的计算机访问网站时，数据包首先会发送到办公室网络路由器。路由器查找标头数据包并确定数据包的目的地。然后，它查找其内部表并将数据包转发到网络内部的下一个路由器或另一台设备，例如打印机。

3.路由算法:

路由算法是实现不同路由协议的软件程序。它们的工作原理是为每条链路分配一个成本数字；成本数字是使用各种网络指标计算的。每台路由器都尝试以最低的成本将数据包转发到下一个最佳链路。

3.静态路由配置实例

即如何让RouterA能够向LAN C发送来自LAN A的数据流——可以通过如下CLI指令完成相应设定：

ip route 192.168.3.0 255.255.255.0 192.168.1.2

这里假设LAN C所属网段为 192.168.3.0/24 ，并且该子网可通过连接至RouterA上具有 IP 地址 192.168.1.2 的端口到达。

4.路由表:

指记录路由信息的表(可以单路由表，也可以多路由表)

当在一台linux机器上要访问一个目标ip:

如果本机有目标ip，则会直接访问本地; 如果本地没有目标ip，则

  1. 用route -n查看路由，如果路由条目里包含了目标ip的网段，则数据包就会从对应路由条目后面的网卡出去
  2. 如果没有对应网段的路由条目，则全部都走网关
  3. 如果网关也没有，则报错：网络不可达

四.什么是 DNS 路由？

域名系统（DNS）将人类可读的域名（例如，www.amazon.com）转换为机器可读的 IP 地址（例如，192.0.2.44）。将此名称信息映射到计算机信息的数据单独存储在 DNS 服务器上。在将数据发送到任何网站之前，路由器必须与 DNS 服务器通信，以确定数据包的确切机器地址。

DNS 服务器通信可能会成为瓶颈，尤其是当许多用户想要同时访问网站时。DNS 路由是指管理与 DNS 服务器通信的各种路由策略和算法。基于延迟的路由和基于地理位置的路由等多种策略有助于管理 DNS 服务器通信负载。
