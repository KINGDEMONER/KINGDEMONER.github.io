一.什么是IP地址

  1. IP地址（Internet Protocol Address）是互联网上每个设备的唯一标识符。本质是一种数字标签，采用特定的二进制编码规则表示(但在显示时常采用点分十进制IPv4 或 冒号十六进制IPv6)，用于在网络环境中定位和区分不同设备。

  2. 作用

  • 网络定位：在网络中精确标识设备位置，确保数据能准确发送到目标设备。
  • 网络通信：源设备在发送数据时，将目的设备的 IP 地址封装在数据包中，网络中的路由器等设备依据此地址进行数据转发，实现设备间通信。

二.公网IP,内网IP

  1. 公网IP

公网IP，公共网络IP地址。它是连接到互联网的设备在全球网络中的唯一标识，如同现实世界中全球通用的“门牌号” 。拥有公网IP的设备能直接与互联网上其他设备进行通信，可被互联网上任何设备访问。例如网站服务器、在线游戏服务器等对外提供服务的设备，都需配备公网IP，以便全球用户能访问其提供的服务。

  1. 内网IP

内网IP，是在局域网（LAN）内部使用的IP地址。它仅在局域网内部具有唯一性，主要用于局域网内设备之间的相互通信。一般情况下，内网IP不能直接与外部互联网进行通信，需借助网络地址转换（NAT）技术，通过路由器将内网IP转换为公网IP，从而实现对外部网络的访问。常见于家庭网络、企业内部办公网络，用于实现设备间的资源共享、文件传输等功能。

三.IP地址分类

  1. **IPv4 **

  • 地址结构：32 位二进制数，为方便使用，常以点分十进制形式表示，即将 32 位二进制数分成 4 组，每组 8 位（1 个字节），每组转换为十进制后用点分隔，如 192.168.1.1。
  • 分类方式：根据地址的第一个字节范围分为 A、B、C、D、E 五类。
    • A 类地址：范围是 0.0.0.0 - 127.255.255.255。适用于大型网络，如跨国企业或大型机构的网络。A类地址 = 网络部分(一1个字节)+主机部分+主机部分+主机部分（3个字节）
    • B 类地址：范围是 128.0.0.0 - 191.255.255.255。常用于中等规模网络，如校园网或企业网。B类地址 = 网络部分+网络部分+主机部分+主机部分
    • C 类地址：范围是 192.0.0.0 - 223.255.255.255。适用于小型网络，如家庭网络或小型企业网络。c类地址 = 网络部分+网络部分+网络部分+主机部分
    • D 类地址：范围是 224.0.0.0 - 239.255.255.255，用于组播通信，可将数据同时发送到一组特定的设备，如视频会议、在线直播等场景。
    • E 类地址：范围是 240.0.0.0 - 255.255.255.255，保留用于科学研究和未来实验性网络。
    • 特殊 IPv4 地址：
      • 网络地址：主机地址全为 0 的 IP 地址代表网络本身，如 192.168.1.0 表示一个 C 类网络。
      • 广播地址：主机地址全为 1 的 IP 地址用于向网络内所有设备发送数据，如 192.168.1.255 是 192.168.1.0 网络的广播地址。
      • 回环地址：127.0.0.1 是本地回环地址，用于测试本地网络协议栈和进程间通信，发往该地址的数据不会真正出现在网络上。

  1. IPv6

  • 地址结构：128 位二进制数，以冒号十六进制表示，将 128 位二进制数分成 8 组，每组 16 位（2 个字节），每组转换为十六进制后用冒号分隔，如 2001:0db8:85a3:0000:0000:8a2e:0370:7334。为简化表示，可省略每组前导 0，连续的 0 组可用双冒号“::”表示，但在一个地址中只能出现一次，如上述地址可简化为 2001:db8:85a3::8a2e:370:7334。

  • 优势：

    • 地址巨大：解决了 IPv4 地址枯竭问题，理论上可提供约 2¹²⁸个地址。

    • 简化报头格式

    • 增强安全性：集成了 IPsec（IP 安全协议），为网络通信提供加密和认证功能。

    • 自动配置：支持无状态自动配置和有状态自动配置，设备接入网络时可自动获取 IP 地址，简化网络管理。

四.子网掩码与子网划分

  1. 子网掩码
  • 与 IP 地址配合使用，在TCP/IP网络中区分 IP 地址中网络地址和主机地址部分。在 IPv4 中，子网掩码同样是 32 位二进制数，与 IP 地址按位进行逻辑与运算，得网络地址。例如，对于 IP 地址 192.168.1.1 和子网掩码 255.255.255.0，进行逻辑与运算后得到 192.168.1.0，即网络地址。
  • 表示方法：除了点分十进制，还可用“/”加上子网掩码中 1 的位数来表示，如 255.255.255.0 可表示为 /24，因为其前 24 位为 1。
  1. 子网划分
  • 原理：将一个大的网络划分成多个较小的子网，通过借用主机地址位来创建子网。例如，对于一个 C 类网络 192.168.1.0/24，若借用 2 位主机地址，则可划分出 2² = 4 个子网，每个子网的主机地址位数减少 2 位，变为 6 位，每个子网可容纳 2⁶ - 2 = 62 台主机（减去网络地址和广播地址）。
  • 目的：
    • 提高 IP 地址利用率。
    • 增强网络管理：便于对不同子网进行独立管理和维护，如网络监控、故障排查等。
    • 增强网络安全性：不同子网间可通过访问控制列表（ACL）等方式进行安全隔离，限制非法访问。
  1. 判断主机是否在同一网段

当两台主机要进行通信时，通过比较它们的网络地址（由IP地址和子网掩码计算得出）来判断它们是否在同一网段。如果网络地址相同，则它们在同一网段，可以直接通信；如果不同，则需要通过路由器进行转。

五.IP 地址分配方式

  1. 静态分配
  • 定义：网络管理员手动为设备分配固定的 IP 地址，设备每次接入网络都使用该固定地址。
  • 适用场景：常用于服务器、网络打印机等需要固定 IP 地址以便其他设备访问的设备。例如，企业内部的文件服务器，为方便员工随时访问。
  • 配置方法：在设备的网络设置中，手动输入 IP 地址、子网掩码、默认网关和 DNS 服务器地址等信息。
  1. 动态分配
  • 定义：通过动态主机配置协议（DHCP）服务器自动为设备分配 IP 地址。设备接入网络时，向 DHCP 服务器发送请求，服务器从预先配置的地址池中分配一个可用的 IP 地址给该设备，并指定租约期限。
  • 适用场景：广泛应用于家庭网络、办公网络等设备数量较多且经常变动的场景。例如，办公室中的员工笔记本电脑，每次接入网络时自动获取 IP 地址，方便快捷。
  • 工作流程：
    • 发现阶段：设备发送 DHCP Discover 广播消息，寻找 DHCP 服务器。
    • 提供阶段：DHCP 服务器接收到 Discover 消息后，向设备发送 DHCP Offer 消息，包含可分配的 IP 地址、子网掩码、租约期限等信息。
    • 请求阶段：设备选择一个 Offer 并发送 DHCP Request 消息，请求使用该 IP 地址。
    • 确认阶段：DHCP 服务器收到 Request 消息后，发送 DHCP ACK 消息，确认设备可使用该 IP 地址，完成分配过程。

六.IP 路由原理

  1. 路由概念
  • 定义：数据在网络中从源设备到目的设备传输时，需要经过多个网络节点（如路由器）的转发，这个过程称为路由。路由器根据路由表中的信息，决定数据包的转发路径。
  1. 路由表
  • 构成：路由器维护的一个表格，包含目的网络地址、子网掩码、下一跳地址、度量值等信息。
  • 目的网络地址：表示数据包要到达的目标网络。
  • 子网掩码：用于确定目的网络地址的范围。
  • 下一跳地址：指数据包转发的下一个路由器接口的 IP 地址。
  • 度量值：衡量到达目的网络的代价，常见的度量值有跳数、带宽、延迟等。不同的路由协议使用不同的度量值计算方式，路由器根据度量值选择最优路径。

七.IP 与网络安全

  1. IP 欺骗
  • 原理：通过伪造 IP 地址，冒充合法设备向目标设备发送数据包，从而获取目标设备的信任或干扰网络正常通信。例如，攻击者伪造一个信任主机的 IP 地址，向目标服务器发送恶意请求，试图获取敏感信息或执行恶意操作。
  • 防范措施：
    • 使用加密技术：如 IPsec 协议，对数据包进行加密和认证，确保数据来源的真实性和完整性。
    • 配置访问控制列表（ACL）：在路由器、防火墙等设备上配置 ACL，限制特定 IP 地址的访问，只允许合法 IP 地址的数据包通过。
    • 检测异常流量：通过入侵检测系统（IDS）或入侵防范系统（IPS）监测网络流量，发现异常的 IP 地址行为并及时报警或阻断。
  1. 防火墙与 IP 规则
  • 防火墙：一种网络安全设备，位于内部网络与外部网络之间，根据预设的安全策略对进出网络的流量进行过滤和控制。
  • IP 规则：防火墙的核心组成部分，通过定义源 IP 地址、目的 IP 地址、端口号、协议等条件，决定是否允许数据包通过。例如，防火墙可配置规则允许内部网络的设备访问外部 Web 服务器（目的 IP 地址为 Web 服务器地址，目的端口为 80 或 443，协议为 TCP），同时阻止外部未经授权的设备访问内部网络的敏感服务器。
  • 配置原则：遵循最小化原则，只开放必要的 IP 地址和端口，关闭不必要的服务和端口，以降低网络安全风险。同时，定期更新防火墙规则，以应对新出现的安全威胁。
