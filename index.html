一.http概念

定义：

http(超文本传输)协议: 是一种用于分布式、协作式和超媒体信息系统的应用层协议。HTTP是万维网的数据通信的基础。

概述：

HTTP是一个客户端终端（用户）和服务器端（网站）请求和应答的标准（TCP）。通过使用网页浏览器、网络爬虫或者其它的工具，客户端发起一个HTTP请求到服务器上指定端口（默认端口为80）。我们称这个客户端为用户代理程序（user agent）。应答的服务器上存储着一些资源，比如HTML文件和图像。我们称这个应答服务器为源服务器（origin server）。在用户代理和源服务器中间可能存在多个“中间层”，比如代理服务器、网关或者隧道（tunnel）。

HTTP协议中，并没有规定必须使用它或它支持的层，HTTP可以在任何互联网协议上，或其他网络上实现。HTTP假定其下层协议提供可靠的传输。因此，任何能够提供这种保证的协议都可以被其使用。所以其在TCP/IP协议族使用TCP作为其传输层。

二.HTTP工作原理：

  1. 客户端与服务器建立连接：一个HTTP客户端，通常是浏览器，与Web服务器的HTTP端口建立一个TCP套接字连接。
  2. 发送HTTP请求：通过TCP套接字，客户端向Web服务器发送一个文本的请求报文。
  3. 服务器接受请求并返回HTTP响应：Web服务器解析请求，定位请求资源。服务器将资源复本写到TCP套接字，由客户端读取。
  4. 释放连接TCP连接：若connection 模式为close，则服务器主动关闭TCP连接，客户端被动关闭连接，释放TCP连接；若connection 模式为keepalive，则该连接会保持一段时间，在该时间内可以继续接收请求;
  5. 客户端浏览器解析HTML内容：
    • 解析状态行，查看表明请求是否成功的状态代码；
    • 解析每一个响应头，响应头告知以下为若干字节的HTML文档和文档的字符集
    • 户端浏览器读取响应数据HTML，根据HTML的语法对其进行格式化，并在浏览器窗口中显示。
  6. 客户端与服务器关闭连接。

三.HTTP请求

请求方法

  1. TTP1.0 定义了三种请求方法： GET, POST 和 HEAD方法。
    • GET 请求指定的页面信息，并返回实体主体。
    • HEAD 类似于 GET 请求，只不过返回的响应中没有具体的内容，用于获取报头，可用于检查某一资源在向其提交GET请求前是否存在
    • POST 向指定资源提交数据进行处理请求（例如提交表单或者上传文件）。数据被包含在请求体中。POST 请求可能会导致新的资源的建立和/或已有资源的修改。：POST方法也与GET方法相似，但最大的区别在于，GET方法没有请求内容，而POST是有请求内容的。
  2. HTTP1.1 新增了五种请求方法：OPTIONS、PUT、PATCH、DELETE、TRACE 和 CONNECT 方法。
    • PUT：PUT方法用于请求服务器把请求中的实体存储在请求资源下，如果请求资源已经在服务器中存在，那么将会用此请求中的数据替换原先的数据。向服务器上传指定的资源。

四.HTTP报文结构

HTTP报文由从客户机到服务器的请求和从服务器到客户机的响应构成：

  • 请求：由请求行，请求头，请求体组成
  • 请求行：包含请求方法、路径、版本号，请求头为多个key-value数据，请求正文包含一些请求的数据
  • 响应：由响应行，响应头，响应体组成
  • 响应行：包含状态码，状态码描述，版本号，响应头为多个key-value数据，响应正文包含一些响应的数据

五.HTTPS协议

HTTPS协议是在HTTP的基础上增加了一个SSL外壳，HTTPS运行在SSL上，SSL运行在TCP上，对数据的加密工作就是在SSL上完成的。

其保证安全性的做法是通过证书验证和对信息混合加密的方式：

混合加密技术：结合对称加密与非对称加密
服务端生成私钥，再通过私钥生成公钥，然后将公钥放在证书中颁发给客户端
使用公钥和私钥以非对称方式加密生成密钥
客户端接下来的传输数据中，都会用密钥以对称方式对信息加密，再传输给服务端

提到的公钥和私钥，我们规定用公钥加密的内容必须用私钥才能解开；同样，私钥加密的内容只有公钥能解开。所以HTTPS传输数据是用被密钥加密的密文和用公钥加密的私钥来保证数据安全的

六.状态码

1.五种状态码：

  • 1xx：信息提示，表示请求已被成功接收，继续处理。
  • 2xx：请求被成功提交。
  • 3xx：客户端被重定向到其他资源。
  • 4xx：客户端错误状态码，格式错误或者不存在资源。
  • 5xx：描述服务器内部错误。

2.常见的状态码描述如下：

  • 200：客户端请求成功，是最常见的状态。
  • 302：重定向。
  • 404：请求资源不存在，是最常见的状态。
  • 400：客户端请求有语法错误，不能被服务器所理解。
  • 401：请求未经授权。
  • 403：服务器收到请求，但是拒绝提供服务。
  • 500：服务器内部错误，是最常见的状态。
  • 503：服务器当前不能处理客户端的请求。

一.什么是IP地址

  1. IP地址（Internet Protocol Address）是互联网上每个设备的唯一标识符。本质是一种数字标签，采用特定的二进制编码规则表示(但在显示时常采用点分十进制IPv4 或 冒号十六进制IPv6)，用于在网络环境中定位和区分不同设备。

  2. 作用

  • 网络定位：在网络中精确标识设备位置，确保数据能准确发送到目标设备。
  • 网络通信：源设备在发送数据时，将目的设备的 IP 地址封装在数据包中，网络中的路由器等设备依据此地址进行数据转发，实现设备间通信。

二.公网IP,内网IP

  1. 公网IP

公网IP，公共网络IP地址。它是连接到互联网的设备在全球网络中的唯一标识，如同现实世界中全球通用的“门牌号” 。拥有公网IP的设备能直接与互联网上其他设备进行通信，可被互联网上任何设备访问。例如网站服务器、在线游戏服务器等对外提供服务的设备，都需配备公网IP，以便全球用户能访问其提供的服务。

  1. 内网IP

内网IP，是在局域网（LAN）内部使用的IP地址。它仅在局域网内部具有唯一性，主要用于局域网内设备之间的相互通信。一般情况下，内网IP不能直接与外部互联网进行通信，需借助网络地址转换（NAT）技术，通过路由器将内网IP转换为公网IP，从而实现对外部网络的访问。常见于家庭网络、企业内部办公网络，用于实现设备间的资源共享、文件传输等功能。

三.IP地址分类

  1. **IPv4 **

  • 地址结构：32 位二进制数，为方便使用，常以点分十进制形式表示，即将 32 位二进制数分成 4 组，每组 8 位（1 个字节），每组转换为十进制后用点分隔，如 192.168.1.1。
  • 分类方式：根据地址的第一个字节范围分为 A、B、C、D、E 五类。
    • A 类地址：范围是 0.0.0.0 - 127.255.255.255。适用于大型网络，如跨国企业或大型机构的网络。A类地址 = 网络部分(一1个字节)+主机部分+主机部分+主机部分（3个字节）
    • B 类地址：范围是 128.0.0.0 - 191.255.255.255。常用于中等规模网络，如校园网或企业网。B类地址 = 网络部分+网络部分+主机部分+主机部分
    • C 类地址：范围是 192.0.0.0 - 223.255.255.255。适用于小型网络，如家庭网络或小型企业网络。c类地址 = 网络部分+网络部分+网络部分+主机部分
    • D 类地址：范围是 224.0.0.0 - 239.255.255.255，用于组播通信，可将数据同时发送到一组特定的设备，如视频会议、在线直播等场景。
    • E 类地址：范围是 240.0.0.0 - 255.255.255.255，保留用于科学研究和未来实验性网络。
    • 特殊 IPv4 地址：
      • 网络地址：主机地址全为 0 的 IP 地址代表网络本身，如 192.168.1.0 表示一个 C 类网络。
      • 广播地址：主机地址全为 1 的 IP 地址用于向网络内所有设备发送数据，如 192.168.1.255 是 192.168.1.0 网络的广播地址。
      • 回环地址：127.0.0.1 是本地回环地址，用于测试本地网络协议栈和进程间通信，发往该地址的数据不会真正出现在网络上。

  1. IPv6

  • 地址结构：128 位二进制数，以冒号十六进制表示，将 128 位二进制数分成 8 组，每组 16 位（2 个字节），每组转换为十六进制后用冒号分隔，如 2001:0db8:85a3:0000:0000:8a2e:0370:7334。为简化表示，可省略每组前导 0，连续的 0 组可用双冒号“::”表示，但在一个地址中只能出现一次，如上述地址可简化为 2001:db8:85a3::8a2e:370:7334。

  • 优势：

    • 地址巨大：解决了 IPv4 地址枯竭问题，理论上可提供约 2¹²⁸个地址。

    • 简化报头格式

    • 增强安全性：集成了 IPsec（IP 安全协议），为网络通信提供加密和认证功能。

    • 自动配置：支持无状态自动配置和有状态自动配置，设备接入网络时可自动获取 IP 地址，简化网络管理。

四.子网掩码与子网划分

  1. 子网掩码
  • 与 IP 地址配合使用，在TCP/IP网络中区分 IP 地址中网络地址和主机地址部分。在 IPv4 中，子网掩码同样是 32 位二进制数，与 IP 地址按位进行逻辑与运算，得网络地址。例如，对于 IP 地址 192.168.1.1 和子网掩码 255.255.255.0，进行逻辑与运算后得到 192.168.1.0，即网络地址。
  • 表示方法：除了点分十进制，还可用“/”加上子网掩码中 1 的位数来表示，如 255.255.255.0 可表示为 /24，因为其前 24 位为 1。
  1. 子网划分
  • 原理：将一个大的网络划分成多个较小的子网，通过借用主机地址位来创建子网。例如，对于一个 C 类网络 192.168.1.0/24，若借用 2 位主机地址，则可划分出 2² = 4 个子网，每个子网的主机地址位数减少 2 位，变为 6 位，每个子网可容纳 2⁶ - 2 = 62 台主机（减去网络地址和广播地址）。
  • 目的：
    • 提高 IP 地址利用率。
    • 增强网络管理：便于对不同子网进行独立管理和维护，如网络监控、故障排查等。
    • 增强网络安全性：不同子网间可通过访问控制列表（ACL）等方式进行安全隔离，限制非法访问。
  1. 判断主机是否在同一网段

当两台主机要进行通信时，通过比较它们的网络地址（由IP地址和子网掩码计算得出）来判断它们是否在同一网段。如果网络地址相同，则它们在同一网段，可以直接通信；如果不同，则需要通过路由器进行转。

五.IP 地址分配方式

  1. 静态分配
  • 定义：网络管理员手动为设备分配固定的 IP 地址，设备每次接入网络都使用该固定地址。
  • 适用场景：常用于服务器、网络打印机等需要固定 IP 地址以便其他设备访问的设备。例如，企业内部的文件服务器，为方便员工随时访问。
  • 配置方法：在设备的网络设置中，手动输入 IP 地址、子网掩码、默认网关和 DNS 服务器地址等信息。
  1. 动态分配
  • 定义：通过动态主机配置协议（DHCP）服务器自动为设备分配 IP 地址。设备接入网络时，向 DHCP 服务器发送请求，服务器从预先配置的地址池中分配一个可用的 IP 地址给该设备，并指定租约期限。
  • 适用场景：广泛应用于家庭网络、办公网络等设备数量较多且经常变动的场景。例如，办公室中的员工笔记本电脑，每次接入网络时自动获取 IP 地址，方便快捷。
  • 工作流程：
    • 发现阶段：设备发送 DHCP Discover 广播消息，寻找 DHCP 服务器。
    • 提供阶段：DHCP 服务器接收到 Discover 消息后，向设备发送 DHCP Offer 消息，包含可分配的 IP 地址、子网掩码、租约期限等信息。
    • 请求阶段：设备选择一个 Offer 并发送 DHCP Request 消息，请求使用该 IP 地址。
    • 确认阶段：DHCP 服务器收到 Request 消息后，发送 DHCP ACK 消息，确认设备可使用该 IP 地址，完成分配过程。

六.IP 路由原理

  1. 路由概念
  • 定义：数据在网络中从源设备到目的设备传输时，需要经过多个网络节点（如路由器）的转发，这个过程称为路由。路由器根据路由表中的信息，决定数据包的转发路径。
  1. 路由表
  • 构成：路由器维护的一个表格，包含目的网络地址、子网掩码、下一跳地址、度量值等信息。
  • 目的网络地址：表示数据包要到达的目标网络。
  • 子网掩码：用于确定目的网络地址的范围。
  • 下一跳地址：指数据包转发的下一个路由器接口的 IP 地址。
  • 度量值：衡量到达目的网络的代价，常见的度量值有跳数、带宽、延迟等。不同的路由协议使用不同的度量值计算方式，路由器根据度量值选择最优路径。

七.IP 与网络安全

  1. IP 欺骗
  • 原理：通过伪造 IP 地址，冒充合法设备向目标设备发送数据包，从而获取目标设备的信任或干扰网络正常通信。例如，攻击者伪造一个信任主机的 IP 地址，向目标服务器发送恶意请求，试图获取敏感信息或执行恶意操作。
  • 防范措施：
    • 使用加密技术：如 IPsec 协议，对数据包进行加密和认证，确保数据来源的真实性和完整性。
    • 配置访问控制列表（ACL）：在路由器、防火墙等设备上配置 ACL，限制特定 IP 地址的访问，只允许合法 IP 地址的数据包通过。
    • 检测异常流量：通过入侵检测系统（IDS）或入侵防范系统（IPS）监测网络流量，发现异常的 IP 地址行为并及时报警或阻断。
  1. 防火墙与 IP 规则
  • 防火墙：一种网络安全设备，位于内部网络与外部网络之间，根据预设的安全策略对进出网络的流量进行过滤和控制。
  • IP 规则：防火墙的核心组成部分，通过定义源 IP 地址、目的 IP 地址、端口号、协议等条件，决定是否允许数据包通过。例如，防火墙可配置规则允许内部网络的设备访问外部 Web 服务器（目的 IP 地址为 Web 服务器地址，目的端口为 80 或 443，协议为 TCP），同时阻止外部未经授权的设备访问内部网络的敏感服务器。
  • 配置原则：遵循最小化原则，只开放必要的 IP 地址和端口，关闭不必要的服务和端口，以降低网络安全风险。同时，定期更新防火墙规则，以应对新出现的安全威胁。

tcp/ip基础

[TOC]

一.tcp/ip的由来

2次世界大战后，美苏冷战，美国国防部想开发一个在恶劣情况下两台计算机仍然可以互相通讯的网络;
1969，阿帕网主要用于军事，同时，民间机构也在研究自己专属的网络，当民间的网络分支越来越多，想法萌生： 将小的局域网扩大，互相通讯。
1972，华盛顿召开会议，决定制定一套通用的网络通讯协议；
1974，美国国防部推出tcp/ip的通讯协议，免费共享出来，开源。
慢慢的，民间机构也使用tcp/ip协议，渐渐形成了英特网

二.tcp/ip是什么？

TCP/IP（传输控制协议/网间协议）是一种网络通信协议，它规范了网络上的所有通信设备，尤其是一个主机与另一个主机之间的数据往来格式以及传送方式。

TCP/IP 是利用IP进行通信时所必须用到的协议群的统称，它不是一个独立的协议，而是一组协议（tcp[传输控制协议]，ip[Internet协议]，udp[用户数据包协议]，arp[地址解析协议]，rarp[反地址解析协议]等协议）。TCP/IP协议族提供了点对点的连结机制，并且将传输数据帧的封装、寻址、传输、路由以及接收方式，都予以标准化。

三.TCP/IP分层架构

TCP/IP协议与七层ISO模型的对应关系，大致如下图所示：

TCP/IP协议是Internet互联网最基本的协议，一定程度上参考了七层ISO模型。OSI模型共有七层，但是这显然是有些复杂的，在TCP/IP协议中，七层被简化为了四个层次。TCP/IP模型中的各种协议，依其功能不同，被分别归属到这四层之中，常被视为是简化过后的七层OSI。

  • 应用层：为网络应用提供服务，如HTTP用于网页浏览、FTP用于文件传输、SMTP用于邮件发送等.
  • 传输层：主要有TCP和UDP协议。TCP提供可靠的、面向连接的数据传输服务；UDP提供无连接、不可靠但高效的数据传输服务，适用于实时性要求高的应用.
  • 网络层：核心是IP协议，负责为数据包选择路由，将数据包从源地址传输到目的地址。此外还有ARP、RARP、ICMP等协议，分别用于地址解析和网络控制等功能.
  • 网络接口层：负责数据帧的发送和接收，涉及实际的网络硬件和链路，如以太网、Wi-Fi等.

四. 协议基础

  • IP协议：为网络中的设备分配唯一的IP地址，进行数据包的路由、分片和重组，使数据能够在不同网络间传输.
  • TCP协议：在IP协议基础上，通过建立连接、数据传输、流量控制和拥塞控制等机制，确保数据准确无误地按顺序从源地址传输到目的地址.
  • UDP协议：无连接、不可靠的传输层协议，不保证数据包的到达顺序和正确性，但具有传输速度快的特点，适用于实时视频、音频传输和网络游戏等对实时性要求高的场景.

五. 数据封装与解封装

数据发送时，从应用层开始依次向下封装，每层添加相应的头部或尾部信息，形成最终的数据包在网络中传输；接收方则从网络接口层开始依次向上解封装，提取出原始数据.

六.地址解析

​ 包括域名解析和ARP解析。域名解析将域名转换为对应的IP地址，以便在网络中找到目标设备；ARP解析则用于将IP地址转换为MAC地址，以实现数据链路层的通信.

七.tcp/ip总结

通俗讲，tcp/ip将计算机通讯划分成四个层次，并规定了每个层次干什么有什么规定，它让不同网络设备互联，确保数据准确传输，支撑多样网络应用，推动互联网发展。

一. OSI参考模型

(一)网络七层模型

​ OSI（Open System Interconnection开放系统互连）七层网络模型称为开放式系统互联参考模型 ，是一种逻辑上的定义规范，把网络从逻辑上分为了7层。每一层都有相关、相对应的物理设备，比如路由器，交换机。

　　OSI七层模型是一种框架性的设计方法，建立七层模型的主要目的是为解决异种网络互连时所遇到的兼容性问题，其最主要的功能是帮助不同类型的主机实现数据传输。它的最大优点是将服务、接口和协议这三个概念明确地区分开来，通过七个层次化的结构模型使不同的系统不同的网络之间实现可靠的通讯。

各层介绍:

1.物理层: 将数字信号转换为物理信号以进行传输，物理层的协议主要包括电压标准、传输介质、接口规范等。（比特流与电子信号的切换）

2.数据链路层：负责数据传输和错误检测，定义了电信号的分组方式。

以太网协议：

早期 , 公司都有自己的分组方式，后来形成了统一的标准，即以太网协议;

　　ethernet规定

  • 一组电信号构成一个数据包，叫做‘帧’
  • 每一数据帧分成：报头head和数据data两部分
mac地址：

　　　　head中包含的源和目标地址由来：ethernet规定接入internet的设备都必须具备网卡，发送端和接收端的地址便是指网卡的地址，即mac地址

　　　　mac地址：每块网卡出厂时都被烧制上一个世界唯一的mac地址，长度为48位2进制，通常由12位16进制数表示（前六位是厂商编号，后六位是流水线号）

​ 3. 网络层：数据路由和分组传输；引入一套新的地址用来区分不同的广播域／子网，这套地址即网络地址。

​ 4.传输层：建立端口到端口的通信；网络层的ip区分子网，以太网层的mac帮我们找到主机，那如何标识这台主机上的应用程序？答案就是端口，端口即应用程序与网卡关联的编号。

端口范围：0-65535，系统占用的端口：0-1023

#ip 确定唯一一台机器
#端口 确定唯一的一个程序
# ip+ 端口 找到唯一机器上的一个程序

传输方式：tcp/udp协议

​ 5.应用层：直接面向用户，处理特定应用程序的协议与数据的交互

​ 6.表示层：位于应用层下方，负责处理数据的表示、转换、加密，让不同系统能有效通信。

​ 7.会话层：处传输层和表示层间，建立、管理、中止对话。对应主机进程，指本地主机与远程主机正在进行的会话。

(二).区别:

​ 网络五层模型和七层模型都用于解析网络通信，七层模型由下至上分为：物理层、数据链路层、网络层、传输层、会话层、表示层、应用层。五层模型把七层模型的会话层、表示层归并入应用层，自下而上是物理层、数据链路层、网络层、传输层、应用层。

(三)总结：

​ OSI七层就是涉及各种协议，加工一些格式来控制通信流程到的方式。

二.网络通信实现

实现网络通信，每台主机需具备四点

  • 本机的IP地址
  • 子网掩码
  • 网关的IP地址
  • DNS的IP地址

　　获取这四要素有两种方式

　　　　1.静态获取：手动配置

　　　　2.动态获取：通过dhcp获取

以太网头 ip头 udp头 dhcp数据包

​ （1）”以太网标头”，设置本机(发出方)的MAC地址和接收方（DHCP服务器）的MAC地址。前者就是本机网卡的MAC地址，后者这时不知道，就填入一个广播地址：FF-FF-FF-FF-FF-FF。

　　（2）”IP标头”，设置发出方的IP地址和接收方的IP地址。这时，对于这两者，本机都不知道。于是，发出方的IP地址就设为0.0.0.0，接收方的IP地址设为255.255.255.255。

　　（3）”UDP标头”，设置发出方的端口和接收方的端口。这是DHCP协议规定好的，发出方端口是68，接收方是67。

​ 数据包构造完成后发出，以太网是广播发送，同一个子网络的每台计算机都收到了这个包。而接收方的MAC地址是FF-FF-FF-FF-FF-FF，看不出是发给谁的，所以每台收到这个包的计算机，必须分析包中的IP地址，才能确定是不是发给自己的。当看到发出方IP地址是0.0.0.0，接收方是255.255.255.255，于是DHCP服务器知道”这个包是发给我的”，而其他计算机就可以丢弃这个包。

​ 之后，DHCP服务器读出包中数据，分配好IP地址，发送回去一个”DHCP响应”数据包。这个响应包的结构也是类似的，以太网标头的MAC地址是双方的网卡地址，IP标头的IP地址是DHCP服务器的IP地址（发出方）和255.255.255.255（接收方），UDP标头的端口是67（发出方）和68（接收方），分配给请求端的IP地址和本网络的具体参数包含在Data部分。

　　新加入的计算机收到这个响应包，就知道了自己的IP地址、子网掩码、网关地址、DNS服务器等参数。
